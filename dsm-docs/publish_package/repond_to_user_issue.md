# Responding to User Issues

## Overview

Post-publication support is essential for package success. Prompt and courteous support can provide a better experience for users, which can result in more downloads and more positive online reviews.

## Support Responsibilities

Synology's technical team handles downloading, installation, and payment issues. Package developers are responsible for:

- Feature requests
- Usage questions
- Compatibility inquiries

## Recommended Actions After Publishing

1. **Establish Support Channels**: Provide links to support resources and create forums or mailing lists for user engagement.

2. **Email Support**: Display an appropriate support contact on your package detail page and respond promptly to user inquiries.

3. **Issue Management**: Acknowledge bugs, implement fixes, and maintain transparency by listing known issues on your package details page.

4. **Update Strategy**: Release updates regularly while maintaining quality standards to avoid excessive notifications that frustrate users.

5. **Release Notes**: Include a summary with each update. Users will read it and appreciate that you are serious about improving the quality of your package.

## Key Takeaway

Responsive developer support directly correlates with user engagement, positive reviews, and higher download rates for Synology packages.
