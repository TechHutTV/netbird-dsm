# Submitting the Package for Approval

## Key Requirements

Before publishing in Package Center, developers must ensure their package meets review standards. It should meet the package review items as detailed in the publication review section.

## Package Pricing Model

An important consideration: packages designated as free cannot be converted to paid later. However, paid packages may be changed to free status at any time. This is a permanent decision for free packages.

## Promotional Materials

Screenshots are mandatory for publication and serve as critical marketing assets. Use high-quality screenshots to showcase your package or brand and consider professional production.

## Submission Process

Developers submit via the Synology website developer application portal. Required documentation includes:
- Package version confirmation
- Download link
- Package description
- Change log detailing updates
- Website link and support email
- Acknowledgment of Developer Distribution Agreement and Terms of Service

## Verification Timeline

The review process involves four major phases:

1. Package and release note receipt
2. Script verification
3. Functional testing across DSM versions and device models
4. Package Center release

Testing may require operation manuals and test scenarios. It is strongly recommended to perform QC verification before submission to expedite the process.
